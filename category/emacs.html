Emacs Rocks - Part 1

Einleitung

Wer sich mit Freier Software beschäftigt, hat vielleicht schon einmal von Emacs gehört. Falls nicht, es handelt sich um einen sehr flexiblen Texteditor. So flexibel, dass manche ihr Betriebsystem zum Bootloader degradieren, dass nur dazu da ist Emacs zu starten.

Leider ist die Lernkurve relativ steil was es Neulingen oft erschwert einen Zugang zu finden. Außerdem gibt es viele alternative Editoren und die meisten sind für Einsteiger zugänglicher als Emacs. Aber wie in dieser Artikelreihe gezeigt werden soll, zahlt es auf lange Sicht mehr als aus, die Benutzung von Emacs zu erlernen.

Motivation

Besonders für Programmierer gibt es eine Fülle von Angeboten an unterschiedlichen Editoren. Oft gibt es sogar Editoren die speziell zur Verwendung einer bestimmten Programmiersprache angeboten werden. Der Nachteil ist, dass man beim Wechsel zu einer anderen Programmiersprache, eventuell einen anderen Editor benutzen muss, da der Support der anderen Sprache in dem verwendeten Editor unter Umständen mangelhaft ist.

Hinzu kommt, dass viele Editoren Tastenkürzel anbieten um bestimmte oft genutzte Funktionen schneller ausführen zu können ohne sich immer wieder durch die selben Menüs klicken zu müssen. Wenn man Glück hat dann bieten die verschiedenen Editoren die man nutzt eine Anpassung dieser Kürzel an, so dass man in verschiedenen Editoren die gleichen Tastenkürzel verwenden kann. Dies ist jedoch meistens nicht der Fall oder man kann nur bestimmte Funktionen mit Tastenkürzeln belegen.

Weiterhin wäre es doch schön, wenn man dieselben Tastenkürzel die man im Editor verwendet auch beim schreiben von Emails oder bei der Eingabe von Befehlen auf der Kommandozeile nutzen könnte.

Neben den Editoren gibt es Programme wie AutoKey welche dem Nutzer die Möglichkeit bieten die Ausfühung beliebiger Aktionen am Computer per Tastenkürzel durchzuführen.

Was wäre wenn es eine Software gäbe, die all diese Probleme adressiert und dazu noch freie Software ist?

Emacs to the Rescue

Emacs ist ein sehr modularer Editor und es gibt für fast jede Programmiersprache Pakete, die Emacs um die Unterstützung der Sprache erweitern. Weiterhin gibt es Erweiterungen mit denen sich aus Emacs ein Email-Client, IRC-Client und vieles anderes machen lässt. Im Prinzip ist Emacs ein großer REPL der zur Laufzeit durch ausführen von Code beliebig verändert werden kann. Das geht soweit, dass andere Editoren wie z.B. Vi/Vim in Emacs emuliert werden können, falls man deren Funktionalität gerne unter Emacs verwenden möchte.

Ehe man sich versieht werden Programme wie der Dateimanager, der Terminal Emulator, Emailclient, IRC CLient, Git Client, Feedreader und mehr durch Emacs ersetzt.

Die erste öffentliche Version von GNU Emacs erschien bereits im Jahr 1985 und es existiert eine große Community, welche die Weiterentwicklung von Emacs vorantreibt. Just am 10. April wurde die Version 24.5 von Emacs veröffentlicht. Man kann also getrost davon ausgehen, das der Support die eigene Lebenszeit überdauert, was bedeutet, dass das Gelernte mit hoher Wahrscheinlichkeit ein Leben lang von nutzen bleibt.
